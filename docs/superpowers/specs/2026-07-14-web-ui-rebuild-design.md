# Know-Agent Web UI 全量重构设计

## 1. 背景与目标

当前生产前端位于 `know-agent-ui/`，使用 Next.js、Ant Design、Ant Design X 和 Tailwind。仓库新增的开源参考项目位于 `web-ui/`，使用 Vite、React 18、Tailwind、Radix UI、shadcn/ui 和 Lucide。

本次重构以 `web-ui/` 为唯一新前端主体，完整采用其 UI 与 UX 语言，但只迁移 Know-Agent 当前已有业务能力。迁移完成并通过验收后删除 `know-agent-ui/`。

目标：

- 用 `web-ui/` 的中性设计令牌、单侧栏、会话列表、沉浸式消息区、紧凑控件和响应式行为替换旧界面。
- 保留智能助理、工作流、知识库、认证、SSE、工具审批和会话管理能力。
- 将工作流设计为可扩展一级模块；PPT 生成只是首个注册工作流。
- 不改变现有后端业务接口，除非实现阶段证明确有缺口且单独评审。
- 在功能、构建、测试和视觉验收通过前保留旧前端作为核对基准。

## 2. 范围

### 2.1 保留的功能

- 登录、令牌保存、用户信息、角色显示和退出。
- 智能助理新建会话、历史会话、切换、删除、消息发送、SSE 回复、工具审批、Markdown、复制和停止生成。
- 工作流目录、运行历史、PPT 工作流启动、节点更新、中断补充、恢复运行和结果展示。
- 知识库文档搜索、状态筛选、刷新、上传、分页、详情、删除、切块、向量化和切片查看。
- 当前权限语义、错误响应和 API 认证头。

### 2.2 明确不保留的开源项目功能

- nanobot WebSocket 协议及其客户端状态。
- Apps、Skills、Automations、Settings、Workspace、归档、置顶、重命名、分叉、语音、二维码和用量热力图。
- 与 Know-Agent 当前能力无关的设置页、工具页和后台协议。

### 2.3 非目标

- 不重做后端文档状态机、RAG pipeline 或 PPT graph。
- 不增加新的工作流业务类型。
- 不增加 OSS 临时上传或预上传接口。
- 不在重构期间删除旧前端。

## 3. 技术策略

采用“领域化重构，保留 web-ui 视觉骨架”方案：

- 保留 `web-ui` 的 Vite 工程、Tailwind 主题、Radix/shadcn 基础控件、Lucide 图标、Markdown 渲染和响应式壳。
- 保留并简化 `Sidebar`、消息区、输入器、对话列表、Dialog、Dropdown、Tooltip 等视觉组件。
- 删除 nanobot 的业务状态、API、WebSocket 和无关页面。
- 按 `assistant`、`workflows`、`knowledge`、`auth` 四个业务域重建状态、hooks、API 和页面。
- 视觉组件不直接调用 API；域 hooks 负责请求、SSE、取消、错误和缓存状态。

建议的目录边界：

```text
web-ui/src/
  app/
    AppShell.tsx
    routes.tsx
  components/
    shell/
    chat/
    workflows/
    knowledge/
    ui/
  features/
    auth/
    assistant/
    workflows/
    knowledge/
  hooks/
  lib/
    api/
    sse/
    formatting/
  tests/
```

具体文件名可在实施计划中按现有代码拆分，但不得重新堆回单一超大 `App.tsx`。

## 4. 信息架构

### 4.1 单侧栏

使用已确认的单侧栏方案：

- 品牌区显示 Know-Agent。
- 一级菜单固定为“智能助理 / 工作流 / 知识库”。
- 一级菜单以下是当前模块的上下文列表。
- 底部显示当前用户与退出入口。
- 桌面可折叠；移动端使用抽屉。

上下文列表：

- 智能助理：助理会话历史。
- 工作流：当前工作流类型的运行历史。
- 知识库：不在侧栏放状态筛选，也不重复文档列表。

助理或工作流仍在运行时，会话条目右侧显示 Codex 风格的旋转 `LoaderCircle`。完成、停止或失败后图标消失。动画必须支持 `prefers-reduced-motion`。

### 4.2 路由

```text
/login
/assistant
/assistant/:threadId
/workflows
/workflows/:workflowId
/workflows/:workflowId/:threadId
/knowledge
/knowledge/:documentId
```

刷新页面后必须恢复当前模块和选中资源。未知工作流或文档返回对应目录，不进入空白页。

## 5. 视觉系统

- 直接使用 `web-ui/src/globals.css` 的中性 HSL 令牌和浅色/深色主题能力。
- 默认圆角保持约 7px；不引入大圆角卡片或装饰性渐变。
- 字体使用现有 system font stack，并保留 CJK 行高优化。
- 图标统一使用 Lucide；熟悉操作使用纯图标按钮并提供 Tooltip 和可访问名称。
- 主界面保持低噪声、紧凑、工作台式布局。
- 卡片只用于工作流目录条目和 Dialog，不嵌套卡片。
- 控件、表格和固定工具栏必须有稳定尺寸，动态内容不得造成布局跳动。

## 6. 智能助理

### 6.1 页面结构

- 进入模块后直接打开最近会话；没有会话时显示居中输入器。
- 对话内容使用受限最大宽度，输入器固定在内容底部。
- 用户消息右对齐，AI 与工具消息左对齐。

### 6.2 消息动作

- 用户消息和 AI 消息都支持复制。
- 每条消息显示 `mm:ss` 时间，使用等宽数字。
- 用户消息动作区右对齐；AI 消息动作区左对齐。
- 复制成功使用轻量通知，不改变消息高度。

### 6.3 发送与停止

- 发送使用 Codex 风格的圆形 `ArrowUp` 图标按钮，不显示“发送”文字。
- 生成中按钮切换为方形停止图标。
- 停止按钮外圈显示旋转反馈，参考旧 `know-agent-ui` 中 `Sender loading` 的视觉语义。
- 不在输入器下方显示“正在生成”文字。
- 点击停止通过 `AbortController` 终止当前流；按钮立即恢复发送状态。

### 6.4 SSE 呈现

- 发送后插入无正文的 loading 消息。
- SSE 分片只写入内存缓冲区，不能把半截正文持续写入消息列表。
- 完成时一次性提交正文。
- 中止时保留已确认的最终内容；没有正文时移除 loading 消息。
- 工具审批继续在对话流中显示批准与拒绝操作。

## 7. 工作流

### 7.1 工作流注册表

工作流通过注册表扩展，不在 `AppShell` 中写死 PPT：

```ts
type WorkflowDefinition = {
  id: string;
  graphName: string;
  title: string;
  description: string;
  icon: LucideIcon;
  capabilities: string[];
  createRunAdapter: WorkflowRunAdapter;
};
```

首个条目：

- `id`: `ppt-build`
- `graphName`: `ppt_build`
- 标题：PPT 生成

### 7.2 工作流目录

- 一级菜单先进入目录，不直接进入 PPT。
- 桌面使用 2 至 3 列紧凑卡片；平板和移动端自适应降列。
- 卡片只显示名称、短摘要、能力标签和图标启动入口。
- 后续新增工作流只需注册定义和运行适配器。

### 7.3 工作流运行界面

- 侧栏显示当前工作流类型的运行历史。
- 顶部包含返回目录、运行标题、当前状态和完成节点数。
- 主区将用户需求、节点活动、中断补充、结果和输入器放在同一时间线。
- 节点活动采用 `AgentActivityCluster` 风格的紧凑纵向进度：完成、当前、待开始三种状态。
- PPT 节点顺序为需求理解、资料检索、模板选择、模板分析、提纲、页面结构和渲染。
- 中断时显示内联“需要补充”区域，输入器切换为补充说明语义。
- 完成时显示可访问的结果链接；失败时保留运行历史和重试入口。
- 工作流消息同样支持复制和 `mm:ss` 时间。

## 8. 知识库列表

### 8.1 工具栏

- 左侧依次为紧凑搜索框和状态 Select。
- 搜索框内显示 `Search` 图标。
- 状态 Select 使用 Radix/shadcn 控件，不使用浏览器原生 Select。
- 右侧为紧邻的刷新图标按钮和上传按钮。
- 刷新中图标旋转，并禁用重复刷新。

### 8.2 表格

至少包含标题、知识库类型、上传人、状态、更新时间和操作列。

- 更新时间格式固定为 `yyyy-MM-dd HH:mm:ss`。
- 更新时间不换行并使用等宽数字。
- 操作列使用查看详情和删除图标按钮，均提供 Tooltip。
- 删除必须二次确认。
- 分页包含总数、每页条数、上一页、连续页码、省略号和下一页。
- 首页和末页正确禁用不可用按钮。
- 移动端表格在自身容器内横向滚动，不拉宽页面。

### 8.3 状态颜色

状态必须同时使用文字、圆点、浅色背景和边框，不能只依赖颜色：

| 状态 | 文案 | 颜色语义 |
|---|---|---|
| `UPLOADED` | 已上传 | 中性灰 |
| `CONVERTING` | 转换中 | 琥珀 |
| `CONVERTED` | 已转换 | 蓝 |
| `CHUNKED` | 已切块 | 青 |
| `VECTOR_STORED` | 已向量化 | 绿 |
| `STORED` | 已入库 | 绿 |

若后端新增失败状态，使用红色错误标签，但不混入正常递进链路。

## 9. 文档详情

文档详情使用独立路由，不使用弹窗：

- 顶部提供返回文档列表、标题、描述、下一步处理动作和删除入口。
- 基本信息包含文档 ID、上传人、知识库类型、角色、切片数量、创建时间、更新时间和当前状态。
- 创建和更新时间使用 `yyyy-MM-dd HH:mm:ss`。
- 生命周期显示已上传、转换中、已转换、已切块和已向量化的递进状态。
- `CONVERTED` 状态提供执行切块；`CHUNKED` 状态提供执行向量化。
- DATA_QUERY 根据当前后端语义显示结构化入库终态。
- 文档切片表包含顺序、Chunk ID、状态和正文摘要。
- 切片内容默认限制行数，可展开；切片列表使用完整分页。

## 10. 文档上传 Dialog

### 10.1 上传时机

采用延迟上传：

- 拖入或选择文件后，文件只保存在浏览器本地。
- 只有点击“上传文档”才调用后端并写入 OSS。
- 关闭或取消只清空本地文件、标题和描述，不调用 OSS 删除。
- 不新增临时对象或清理接口。

### 10.2 交互

- 支持点击选择和拖拽文件。
- 支持 PDF、Word、Excel、CSV、TXT 和 Markdown，单次一个文件。
- 选中文件后显示文件名、扩展名、大小和更换文件入口。
- 标题与描述自动使用去掉扩展名的文件名称，用户可继续修改。
- 知识库类型使用 Radix/shadcn Select，箭头距离右边缘约 10 至 12px。
- 角色使用 Radix DropdownMenu CheckboxItem 实现多选下拉，并在触发器内显示已选角色标签。
- 不选择角色表示公开。
- DATA_QUERY 时显示表名字段；DOCUMENT_SEARCH 时不显示。
- 提交中禁止关闭、取消和重复提交。

## 11. 认证与数据流

- `AuthProvider` 保存 token 和用户资料。
- API 客户端统一添加认证头并解析错误。
- `401` 清理本地会话并回到登录页。
- `403` 在当前操作附近提示权限不足。
- 一级模块和资源 ID 进入 URL。
- 助理和工作流历史按 agent/graph 类型分别读取。
- 知识库继续使用现有文档 API，不修改请求字段。
- 各业务域 hooks 管理加载、错误、中断和刷新，视觉组件只收 props 和事件。

## 12. 错误与空状态

- 短反馈使用轻量通知；需要处理的问题放在触发位置附近。
- 网络错误保留当前输入并允许重试。
- SSE 断开时保留用户消息，移除空 loading 消息并显示重试入口。
- 删除失败时恢复原列表状态。
- 上传、切块、向量化期间禁用重复操作，并在对应行显示进行中状态。
- 空会话、空工作流、空文档和空搜索结果使用各自的简洁空状态，不使用营销式说明。

## 13. 测试

### 13.1 单元测试

- API 请求、响应映射和错误解析。
- SSE 事件解析、缓冲和 AbortController 中断。
- 工作流注册表与路由查找。
- 文档状态颜色映射。
- `mm:ss` 和 `yyyy-MM-dd HH:mm:ss` 格式化。

### 13.2 组件测试

- 一级导航、上下文会话列表和运行图标。
- 用户与 AI 消息复制、时间、发送/停止切换。
- 工具审批。
- 工作流目录、节点进度、中断补充和完成结果。
- 文档搜索、状态 Select、刷新、分页、详情和删除。
- 上传拖拽、自动回填、知识库类型 Select、角色多选和取消清理。

### 13.3 集成测试

- 登录与认证失效。
- 助理新会话、历史、SSE、停止和删除。
- PPT 工作流启动、中断、恢复和结果。
- 文档上传、切块、向量化、详情、切片和删除。

### 13.4 视觉与响应式测试

- 桌面和移动端检查侧栏、抽屉、输入器、表格、Dialog 和下拉菜单。
- 检查浅色和深色主题。
- 检查长标题、长文件名、长角色名和长 Markdown。
- 对照 `web-ui` 参考视觉检查字体、边框、圆角、间距、图标和交互状态。

## 14. 验收与切换

删除旧前端前必须同时满足：

1. 当前功能清单逐项通过。
2. 新增交互清单逐项通过。
3. `npm run test` 通过。
4. `npm run build` 通过。
5. 桌面和移动端视觉检查通过。
6. 认证、助理 SSE、停止、审批、PPT 工作流和文档状态链路通过真实后端验证。
7. `web-ui` 成为开发和生产文档中的唯一前端入口。

满足以上条件后删除 `know-agent-ui/`，清理旧前端脚本和文档引用，并再次执行构建与回归测试。
